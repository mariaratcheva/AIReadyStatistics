Data exposed with revision and release status ()
Principle. Every disseminated value should carry explicit release and revision status: its production status (provisional, final, revised), its release or version identifier, the date it was produced, and, where a figure has been superseded, a link to the current authoritative version. Rationale. Official statistics change over time: provisional figures become final, scheduled releases supersede earlier ones, and corrections are occasionally issued. A figure that is correct today may be superseded tomorrow, and an AI system that has retrieved or cached an earlier value cannot detect this unless the architecture makes the status explicit. This status is not optional metadata; for AI consumption it is part of what makes a figure safe to use, because it lets an agent distinguish a current authoritative figure from a stale one and cite the correct version. Constraint. Embargoed data must never be reachable through any route before release.
