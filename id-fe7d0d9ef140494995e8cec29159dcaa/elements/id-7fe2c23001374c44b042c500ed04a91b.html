Decoupled and interoperable AI-facing layers ()
Principle. Distinguish durable dissemination foundations from the technical layer that exposes them to AI. AI-facing services (for example, an MCP service) should build on existing data APIs, metadata APIs, catalogues, and search rather than duplicating them. Rationale. This makes the architecture resilient: if the dominant AI-exposure standard changes, only the thin AI-facing layer needs to evolve. That layer should rely on recognised interoperable standards (MCP is one current example) rather than vendor-specific solutions, reducing lock-in.
